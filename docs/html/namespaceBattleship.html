class  GamesRegister
 GamesRegister es una clase estática que funciona como base de datos de los Game. Se accede a los Game a travez del id. En el momento que se inicia un partida, se crea un Game, y en el momento que finaliza, se elimina su referencia desde la clase GamesRegister, dejando a voluntad del Garbage Collector para que lo elimine cuando lo considere conveniente. Ese es el ciclo de vida de los Game.

class  BaseHandler
 Clase base para implementar el patrón Chain of Responsibility. En ese patrón se pasa un mensaje a través de una cadena de "handlers" que pueden procesar o no el mensaje. Cada "handler" decide si procesa el mensaje, o si se lo pasa al siguiente. Esta clase base implmementa la responsabilidad de recibir el mensaje y pasarlo al siguiente "handler" en caso que el mensaje no sea procesado. interface  IHandler
 Interfaz para implementar el patrón Chain of Responsibility. En ese patrón se pasa un mensaje a través de una cadena de "handlers" que pueden procesar o no el mensaje. Cada "handler" decide si procesa el mensaje, o si se lo pasa al siguiente. Esta interfaz define un atributo para definir el próximo "handler" y una una operación para recibir el mensaje y pasarlo al siguiente "handler" en caso que el mensaje no sea procesado.
